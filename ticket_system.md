ticket_system.md

티켓 생성 시부터 티켓 마감까지, 해당 티켓은 연동되고 같은 티켓을 불러오는 형태로 이뤄지게 해야해. 처음 티켓을 우편실에서 검수를 통해 생성하거나, 문의답변에서 신규 티켓 작성으로 생성하거나 같은 내용을 작성을 해야하고 (우편실에서는 담당자 배정이 가능하고, 관리자급부터 가능 / 문의답변에서는 직원도 가능하고 담당자 배정은 관리자급부터 가능하고)

티켓에서 충전/입금/이체  -   차감(도서구매/물품구매/대행/기타) - 배팅 업무(배팅포인트-일반포인트 전환/배팅) 업무가 처리되게끔 해주고.

티켓의 상태를 접수 -> 배정 -> 처리중(답변중) -> 처리중(답변완료) -> 처리완료(답변완료) -> 마감(마감시 미처리 티켓에서 없어지게.) 마감 후에는 답변보관함으로 옮겨지고 회원 조회(회원 관리에서 회원선택시) 시 불러와지게 

마감 전까지는 문의답변 탭에 가면 자신에게 배정된 티켓이 모두 보여지고, 관리자급 이상은 모든 미처리 티켓이 보여지게.

가장 핵심은 장바구니 시스템을 각 업무 처리에 필요하게 개량해서 적용시키는 것.
이 모든 기능들을 백엔드/프론트엔드를 완벽히 연결해주고 기존 시스템과 연동해줘.
그리고 이 기능들과 겹치는 기존의 기능들을 필요한 정도를 판단해서 완전히 중첩되면 주석처리하고. 보고해줘.
